# Contributing to CoderCorps 🚀

Welcome! CoderCorps is a learning community project dedicated to accelerating software engineering careers through real-world project development, peer review, and credentialing. We are excited that you want to contribute! First-time contributor pull requests are highly welcome.

---

## 🌳 Branch Naming Convention

To keep our repository organized, we follow a standard prefix convention for branch names:

* **Features:** `feature/<short-description>` (e.g., `feature/login-validation`)
* **Bug Fixes:** `fix/<short-description>` (e.g., `fix/auth-cookie-expiry`)
* **Documentation:** `docs/<short-description>` (e.g., `docs/readme-ci-badge`)
* **Refactoring:** `refactor/<short-description>` (e.g., `refactor/api-helper`)

---

## 🛠️ Contribution Workflow

1. **Fork the Repository:** Create a personal copy of the repository.
2. **Clone & Set Up:** Follow the setup instructions in the [README.md](file:///d:/MycoderCorps/README.md).
3. **Create a Branch:** Create a branch following the naming convention above.
4. **Make Changes:** Keep commits small, descriptive, and focused on a single topic.
5. **Run Checks Locally:**
   - **Frontend:** Make sure typescript type-checks, ESLint passes (`npm run lint`), and the application builds successfully (`npm run build`).
   - **Backend:** Ensure python tests pass successfully via `pytest` inside the virtual environment.
6. **Push & Open PR:** Push to your fork and submit a Pull Request to our main branch.

---

## 🤝 Pull Request Guidelines

* Fill out the Pull Request template completely.
* Verify that the GitHub Actions CI pipeline passes.
* Assign a reviewer and address review feedback promptly.
