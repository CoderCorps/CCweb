Part 1 — Mentor Capabilities

The complete technical lead workflow: from project initiation to certificate minting.

MENTOR ROLE
PHASE 01
Initiation & Planning
  • Create Project
  • Define Sprints
  • Create & Assign Tasks
PHASE 02
Active Execution
  • Monitor Standups
  • Submit Feedback
  • Resolve Blockers
  • Workspace Chat
PHASE 03
Code Audit & Review
  • Review Task PR
  • Assign Score 0-100
  • Approve / Revision
  • Trigger Badge Eval
PHASE 04
Completion
  • Review Final Repo
  • Approve Project
  • Mint Certificate
01
Project Initiation & Member Control
Projects reside inside parent Programs (cohorts/semesters) initialized by Admins. Mentors initialize their project workspaces and manage enrollment.
REST API Endpoints
POST/api/v1/projects/
POST/api/v1/projects/{id}/members
DELETE/api/v1/projects/{id}/members/{user_id}
src/app/(platform)/mentor/students/page.tsx
02
Sprint & Task Planning (Kanban Board)
Mentors set up the project's timeline by defining sprints and task requirements. They allocate technical tasks to specific students to organize work across the sprint.
REST API Endpoints
POST/api/v1/projects/{id}/sprints
POST/api/v1/projects/sprints/{sprint_id}/tasks
POST/api/v1/tasks/{id}/assign
PATCH/api/v1/projects/tasks/{task_id}
src/app/(platform)/projects/[id]/page.tsx — Kanban Board
03
Daily Standup Reviewing & Mood Tracking
Every morning, students state their daily checklist. Every evening, they submit a Daily Standup report. Mentors monitor these reports to catch drop-offs and support struggling students.
REST API Endpoints
GET/api/v1/daily/reports
PATCH/api/v1/daily/reports/{id}/feedback
GET/api/v1/daily/reports/missing
src/app/(platform)/mentor/reports/page.tsx
04
Stuck Students & Blocker Resolution
When students hit technical road-blocks, they can place an explicit "Stuck Flag" on an assigned task. Mentors can view, triage, and resolve these flags — or hop in a team chat room to debug in real time.
REST API Endpoints
GET/api/v1/projects/{id}/stuck-flags
PATCH/api/v1/stuck_flags/{id}/resolve
WS/ws/rooms/{project_id}
05
Task-Level Code Reviews & Grading
When a student moves a task to review and submits their work, the mentor performs a line-by-line review of their pull request or code approach.
REST API Endpoints
GET/api/v1/tasks/{id}/submissions
PATCH/api/v1/submissions/{id}/review
System Logic (Auto-Triggered)
  • 1
    Score submission transitions TaskAssignment status to "reviewed"
  • 2
    Triggers in-app Notification to the student immediately
  • 3
    Runs background badge evaluation: evaluate_blocker_crusher() & evaluate_system_architect()
src/app/(platform)/projects/[id]/leaderboard — Submission Deck
06
Project-Level Audits & Certificate Minting
Upon completing all milestones, students submit a final comprehensive project delivery (Live Demo + Production GitHub Repository). Mentors evaluate this final delivery.
REST API Endpoint
PATCH/api/v1/submissions/{id}/review
System Logic on Approval
  • ✓
    Auto-generates a cryptographic Certificate with student name, project title, mentor name, approved timestamp, repo URL, demo URL
  • ✓
    Creates a platform-wide certificate_issued activity feed event
src/app/(platform)/mentor/reviews/page.tsx — Reviews Board

Part 2 — Admin Example Flow

Elena initializes the "Fall 2026 Systems Cohort" and manages platform growth.

ADMIN ROLE
Scenario: Initializing a New Semester Cohort and Overseeing Platform Growth
1
Elena logs into /dashboard as Admin
Full admin command center loads with global KPI metrics: total projects, students enrolled, mentors, pending reviews, and the real-time activity log.
2
Creates Program "Fall 2026 Systems Cohort"
POST/api/v1/programs/
{
  "name": "Fall 2026 Systems Cohort",
  "slug": "fall-2026-systems"
}
Program container created. Ready for Mentor projects.
3
Promotes "Marcus Vance" from Student → Mentor
Elena locates Marcus Vance in the User Management Panel on the dashboard roster and cycles his role using the inline control.
PATCH/api/v1/dashboard/users/{user_id}/role
{ "role": "mentor" }
Marcus now has access to the /mentor workspace and all mentor-gated routes.
4
Reviews Global Activity Audit Feed
GET/api/v1/activity/recent
Renders real-time highlights of merged PRs, certificate issuances, task reviews, and badge unlocks system-wide.

Part 3 — Mentor Example Flow

Marcus Vance leads the "LLM-Powered Code Review Bot" project end-to-end.

MENTOR ROLE
Scenario: Leading the "LLM-Powered Code Review Bot" Project
1
Create Project Workspace
POST/api/v1/projects/
{
  "title": "LLM Review Bot",
  "program_id": 1,
  "max_members": 5
}
Project workspace initialized. Marcus added as lead member automatically.
2
Plan Sprint & Define Tasks
POST/api/v1/projects/{id}/sprints

3
Assign Task to Student "Aria Chen"
POST/api/v1/tasks/{task_id}/assign
{ "user_ids": [aria_user_id] }
4
Daily Mentorship Loop (Morning → Afternoon → Evening)
☀ Morning
GET/api/v1/daily/reports/missing
⚡ Afternoon — Aria raises a Stuck Flag
WS/ws/rooms/{project_id} — Marcus resolves bug in chat
PATCH/api/v1/stuck_flags/{id}/resolve
🌙 Evening — Report Review
GET/api/v1/daily/reports
PATCH/api/v1/daily/reports/{id}/feedback
5
Code Review Audit — Grade Aria's Submission
PATCH/api/v1/submissions/{sub_id}/review
{
  "mentor_score": 95,
  "mentor_feedback": "Clean implementation of the NodeVisitor class."
}
Aria notified. Skill points incremented. Badge evaluation triggered.
6
Graduation & Certification — Final Project Approval
PATCH/api/v1/submissions/{project_sub_id}/review
{
  "status": "approved",
  "feedback": "Full pipeline functioning."
}
Verifiable Certificate auto-generated for Aria! Activity feed event emitted.
